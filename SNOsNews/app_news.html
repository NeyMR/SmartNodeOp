SmartNodeOps Tool Feature Highlights:

Pi Node Installer:

  • One-click installation of Virtual Machine Platform and Windows Subsystem 2 (WSL2).
  • Automatic installation of required software, including Docker 4.12 and the latest Pi Node App (v0.4.5).

Json File Fix:

  • Intelligent repair of Json file issues, automatically resolving Node Not Loading problems commonly encountered in new Pi Node installations.

Local Port Check and Windows Firewall Opener:

  • Obtain Local IP automatically and check local Open Ports.
  • Provides step-by-step instructions to open necessary ports if found closed.
  • Ensures your Pi Node host machine has the required open ports for optimal Pi Node performance.

Pi Node Metrics:

  • Comprehensive display of valuable node information for efficient diagnosis and monitoring.
  • Node Factor Calculator(BareBone still under Adjusments) Calculates Factor Based on resource Availability.
  • Access to error logs and other vital metrics.

Pi Consensus Container metrics:

  • Comprehensive display of valuable Container information for efficient diagnosis and monitoring.
  • container CPU and Memory Usage metrics.

With this initial release, we aim to provide essential tools to get your Pi Node up and running effortlessly. Rest assured, we have implemented an auto-updater, making it quick and simple to stay up-to-date with future feature additions and improvements.

Version 1.0.7.0 Feature Highlights:

Pi Consensus Container Metrics Updates:

  • Docker Volume and Available Space Visualization: This enables a visual representation of Node Disk Usage.
  • Log Size Adjustments: Logs now retain valuable time-framed information while striving to minimize application memory usage. This is an ongoing process to keep the tool lightweight and with minimal impact on system performance during monitoring.
  • Reduced Frequency of CPU and Memory Usage Metrics: These metrics now update every 60 seconds (equivalent to 10 minutes for a full frame).

SmartNodeOps Fixes:

  • Addition of Memory Manager and DataManager: These enhancements improve performance for long-run monitoring. It's recommended to run monitoring for shorter periods while we continue to fine-tune these features. Currently, a 6-hour run uses around 500-600MB of memory, but we're aiming for even lower memory usage for longer monitoring sessions.
  • Performance Updates: More performance optimizations have been implemented in the codebase.
